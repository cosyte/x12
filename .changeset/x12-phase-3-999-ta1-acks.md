Phase 3 acknowledgments: pure-function `parse999` / `build999` (TR3 005010X231A1) and `parseTA1` / `buildTA1` (ASC X12 standard envelope-level). The cosyte ack archetype — the library MECHANICALLY builds the disposition it is told and REFUSES to fabricate an Accept against a non-empty error list — lands here with four typed `ACK_BUILD_ERROR_CODES`: `X12_ACK_ACCEPT_WITH_ERRORS`, `X12_ACK_COUNT_MISMATCH`, `X12_ACK_INVALID_SPEC`, `X12_TA1_ACCEPT_WITH_NOTE`. `parse999` decodes AK1 → AK2 → (IK3 [→ CTX] (IK4 [→ CTX])\*)\* → IK5 → AK9 into the typed `X12Ack999`; legacy `AK3`/`AK4`/`AK5` senders are lenient-accepted on parse (normalized onto X231A1 names) per Postel's Law, while `build999` always emits the X231A1 names. The envelope walker now captures envelope-level TA1 segments verbatim onto `X12Interchange.ta1Segments` (a new additive `readonly Ta1Segment[]` field) — TA1 between ISA and the first GS is recognized as spec-conformant (no `X12_UNEXPECTED_SEGMENT` warning), a TA1 inside an open functional group is still flagged. Code-list registries shipped as frozen string-literal unions for exhaustive narrowing: `X12_ACK_DISPOSITION_CODES` (715), `IK3_SYNTAX_ERROR_CODES` (716, 13 codes), `IK4_SYNTAX_ERROR_CODES` (723, 18 codes), `TA1_ACK_CODES` (I13), `TA1_NOTE_CODES` (I18 `000`–`028`); the `isAcceptDisposition(code)` predicate returns `true` for `A`/`E`/`P` and `false` for `R`/`M`/`W`/`X`. Acks are structurally PHI-free by design — control numbers, segment IDs, position counters, and structural error codes only. The one variable-shape PHI surface, `IK4-04` (`copyOfBadDataElement`), is documented on both the parsed-model and build-spec types as a caller-supplied field that callers SHOULD omit when the offending bytes are PHI; the library NEVER auto-populates it. Six Tier-1 fixtures land (three 999 dispositions + three TA1 dispositions); a fast-check round-trip property exercises `parse999(build999(spec))` 200 runs; a fast-check accept-clean safety property proves the `Accept`-with-errors refusal across 200 runs. Acceptance-grade verify gate green (typecheck + lint + format + coverage ≥ 90 globally + build + attw + exports). No new entries to `WARNING_CODES` or `FATAL_CODES` — those registries stay at the Phase-2-locked sizes of 10 and 4.

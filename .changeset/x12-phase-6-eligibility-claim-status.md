Phase 6 — 271 eligibility + 277 / 277CA claim-status surface. Three new public walkers extract typed models from a parsed transaction set: `get271Eligibility(delimiters, tx)` (TR3 `005010X279A1`), `get277Status(delimiters, tx)` (TR3 `005010X212`), and `get277CADisposition(delimiters, tx)` (TR3 `005010X214`). 277 and 277CA share one internal walk; the two entry points differ only in which `ST-03` implementation-convention reference they admit (`get277CADisposition` accepts only `005010X214`; `get277Status` accepts either). Each returns `undefined` only on a mis-routed call (wrong `ST-01`); every recoverable deviation surfaces as a warning.

**TRN echo is locked as a safety property.** A 271 carries the requesting 270's `TRN-02` trace verbatim on its enclosing subscriber / dependent, and a 277 carries the 276's on its claim, so a provider can re-associate the answer with the request — the walkers never mutate, normalize, or drop it. A round-trip property test asserts byte-for-byte echo across an arbitrary trace grammar.

**Status-code fidelity** for the 277 family: each STC composite (STC-01 / STC-10 / STC-11, C043) decodes into a verbatim CSCC (Claim Status Category Code, X12 source 507) + CSC (Claim Status Code, source 508) + responsible-entity triple; bundled snapshot descriptions resolve when known and codes outside the subset preserve their verbatim value and emit `X12_UNKNOWN_CLAIM_STATUS_CATEGORY` / `X12_UNKNOWN_CLAIM_STATUS`. New bundled, dated code-list snapshots ship alongside the CARC / RARC family: `CLAIM_STATUS_CATEGORY_CODES`, `CLAIM_STATUS_CODES`, and `SERVICE_TYPE_CODES` (with `lookupClaimStatusCategory` / `lookupClaimStatus` / `lookupServiceType`).

HL parent-pointer integrity is enforced through the shared `validateHl` primitive (271 spine `20→21→22→23`; 277 / 277CA spine `20→21→19→22→23`) — a dangling or mis-levelled parent emits `X12_HL_PARENT_MISMATCH` / `X12_HL_PARENT_LEVEL_INVALID` and the walker NEVER silently re-numbers. A 277CA provider-level batch acknowledgment opens a claim on a standalone STC (no TRN). All monetary fields (EB amounts, STC charge / payment, SVC line charge / payment) decode as `X12Decimal`, never `parseFloat`. 13 dogfooded `LoopSpec` artifacts ship through the public `defineLoopSpec()` (7 eligibility + 7 status, with Loop 2200 / 2220 reused across subscriber + dependent branches). Warning registry expanded 18 → 20 (additions-only); both new factories shape-validate the echoed code (H-PHI invariant). Shared `X12Hl` HL primitive exported for the 271 / 277 / 277CA result types. Six synthetic fixtures (271 canonical + dependent; 277 canonical + unknown-status; 277CA canonical batch-ack + HL-orphan quirk), unit tests, plus byte-flip fuzz (never-throw outside the 4 Tier-3 fatals) across every Phase 6 fixture.

Known limitations (deferred): AAA request-validation segments, HSD health-service-delivery detail, and III / LS / LE markers in the 271 are preserved on `tx.segments` but not yet typed; QTY / AMT claim-summary roll-ups in a 277CA Loop 2200 are likewise preserved verbatim, not modelled.

Phase 4 — 835 Healthcare Claim Payment/Advice (ERA), TR3 005010X221A1. The cash-posting surface — money, the consultant ask. `get835(delimiters, tx)` walks a parsed 835 transaction set into the typed `X12Remittance` model: BPR payment header, TRN trace numbers, Loop 1000A / 1000B payer + payee (with address / contact / additional identifiers), Loop 2100 claims (CLP + patient/subscriber/service-provider NM1s, statement-period DTMs, CAS adjustments at claim and service-line scope, MIA/MOA/LQ remarks, REF/AMT supplemental amounts), Loop 2110 service lines (SVC with HCPCS/CPT/NDC/revenue-code/modifier destructuring, service-date DTMs, line-level CAS/REF/AMT/LQ), and PLB provider-level adjustments (multi-pair flattening). All monetary fields decode as the new string-backed `X12Decimal` — `BigInt`-exact arithmetic, **NEVER `parseFloat`**: float representation silently destroys cents at scale, and on an 835 a dropped decimal is the wrong dollar amount in someone's cash post. `X12Decimal` preserves the inbound lexical form for byte-exact round-trip, exposes mathematical equality across scales (`"0.00".equals("0")` → true), and ships `add` / `subtract` / `compareTo` / `abs` / `negate` / `signum` / `isZero` / lossy `toNumber()` plus `fromBigInt(value, scale)` for canonical rendering. Three balance invariants run after the walk per TR3 §1.10.2 — line (`SVC-02 == SVC-03 + Σ(line CAS)`), claim (`CLP-03 == CLP-04 + Σ(all CAS, claim + line)` — the X12 spec balance, with CLP-05 informational only), and top-of-remit (`BPR-02 == Σ(CLP-04) - Σ(PLB amounts)` with PLB amounts carrying the raw EDI sign: positive = take-back). Mismatches emit `X12_835_REMIT_BALANCE_MISMATCH` carrying the invariant label + spec/computed/delta `X12Decimal` strings; the model is **NEVER silently rebalanced**, never echoes patient identifiers in the warning message. Three built-in `LoopSpec` artifacts (`REMIT_835_LOOP_2000`, `REMIT_835_LOOP_2100`, `REMIT_835_LOOP_2110`) ship through the public `defineLoopSpec()` API — the dogfooding gate locked in Phase 2 — plus payer/payee Loop 1000A/B specs as introspection artifacts. Bundled WPC + X12-internal code-list snapshots ship as versioned data artifacts (`CARC` ~30 codes, `RARC` ~15 codes covering both `M`- and `N`-prefix conventions, `CLP_STATUS` 10 dispositions, `CLAIM_ADJUSTMENT_GROUP_CODES` as a frozen 4-value literal union: `CO`/`PR`/`OA`/`PI`); unknown codes preserve the verbatim value on the parsed model AND emit `X12_UNKNOWN_CARC` / `X12_UNKNOWN_RARC` with shape-validated echoes (codes pass `/^[A-Z0-9]{1,5}$/u` or collapse to `(non-spec)`). The `lookupCarc` / `lookupRarc` / `lookupClpStatus` helpers return frozen `{ code, description }` for known codes; the `isClaimAdjustmentGroupCode` predicate narrows inbound strings. The pre-launch `0.0.x` policy says these snapshots ship as **initial subsets**; a Phase 10 `pnpm refresh:code-lists` will regen the full WPC-published lists for the first real publish. Warning registry expanded 10 → 13 (additions-only); fatal registry stays at 4. PHI discipline (H-PHI invariant holds suite-wide): warning messages never echo field VALUES; patient names / member ids / NPIs / claim numbers are held verbatim on the parsed model (consumer-redaction boundary, mirroring `@cosyte/hl7`'s posture) but never routed through warnings or errors. Six fixtures land under `test/fixtures/remit/` — five Tier-1 synthetic spec-clean (`835-medicare-canonical.edi`, `835-multi-claim.edi`, `835-with-plb.edi`, `835-carc-rarc-mix.edi`, `835-imbalance.edi`) and one Tier-2 synthetic Availity-quirk shape (`835-availity-quirk.edi`); the imbalance fixture is deliberately off-by-$10 to prove the warning fires + the model preserves the verbatim amounts. Property tests lock the `X12Decimal` algebra (round-trip + additive identity + commutativity + subtraction-by-addition + negation involution + sign consistency) over 500 runs; the 835 balance property over 100+50 runs; the 835 byte-flip fuzz target across every fixture (300 runs per fixture) proves `get835` never throws outside the 4 Tier-3 fatals. Coverage gates expanded per-directory ≥90 to `parser/` / `loops/` / `transactions/` / `code-lists/`. Verify gate green (typecheck + lint + format + coverage 97.7% stmts / 91.97% branches / 99.24% funcs / 99.38% lines + build + attw + verify:exports). 269 tests total. Known limitations: no 835 *building* yet (Phase 8); bundled CARC/RARC are initial subsets; no payer profile system (Phase 9); CPT/ICD-10/NDC descriptions deliberately NOT bundled (license-gated); `X12Decimal` does not yet expose multiply/divide; `phi-scan` script not yet wired for x12 (the H-PHI property tests provide runtime coverage — an explicit pre-commit phi-scan ships in a future slice).

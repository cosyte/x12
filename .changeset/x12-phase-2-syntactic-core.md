Phase 2 syntactic core: every segment inside a transaction is now decoded into an immutable `X12Segment` carrying its id, raw text, and 1-indexed element array; the verbatim source survives on `X12TransactionSet.rawSegments` for byte-exact round-trip. The `?`-release-character escape (`?~` literal `~`, `?*` literal `*`, `??` literal `?`) is handled by `unescapeRelease` / `escapeRelease` / `splitWithRelease` (zero-dep, lossless round-trip property for any value across any 4-distinct-delimiter set). Dot-path traversal — `getSegmentValue(seg, "03-1")` for composites, `"03[2]"` for 0-indexed repetitions, `"03[2]-1"` for repetition + component — with a matching `getAllSegmentValues` for every-repetition reads. Public `defineLoopSpec()` API ships with structural validation + `LoopSpecDefinitionError` so Phase 3+ transaction extractors author their built-in TR3 loops through the same surface consumers use (the dogfooding gate). Warning registry expanded from 8 → 10 with `X12_DANGLING_RELEASE_CHAR` (unpaired `?` at element/segment end, byte-preserved) and `X12_UNEXPECTED_SEGMENT` (envelope-structural anomalies the Phase 1 walker dropped silently). PHI discipline: `X12_UNEXPECTED_SEGMENT` shape-validates the echoed segment id against `/^[A-Z][A-Z0-9]{1,2}$/` and substitutes `(non-spec)` for anything else, preventing hostile input from leaking PHI into warning messages (mirrors the hl7 H-PHI invariant). New Tier-1 fixture (`syntactic-core-body.edi`) plus property tests for `?`-escape round-trip, segment dot-path, and streaming-decode chunk-invariance (locks the v2 streaming surface as a non-breaking addition).

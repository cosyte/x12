## Decisions Made

- **Used Node.js to satisfy jq-style verification.** The plan's `<verify>` blocks invoke `jq -e`, but `jq` is not installed on this machine and `sudo` is unavailable. All Task 1 + Task 2 acceptance criteria passed under the substitute checks. Reading or running the verify block in CI will require either installing jq (e.g., in Plan 04 CI workflow) or rewriting the verify block to use Node — flagged for Plan 04.
- **`pnpm typecheck` substitute.** The orchestrator's success criterion mentions `pnpm typecheck`, but the `typecheck` script is wired in Plan 03 per the phase plan (Task 2 acceptance criteria of this plan explicitly say `pnpm typecheck` is not required here). To prove the strict tsconfig loads cleanly under the stub regardless, ran `npx --yes -p typescript@5.6.3 tsc --noEmit -p tsconfig.json` — exit code 0. The `npm warn` lines about `auto-install-peers` / `strict-peer-dependencies` are harmless (`npm` doesn't recognize pnpm-specific `.npmrc` keys).
- **Deferred sibling-parity audit.** Without the sibling repo on disk, exact mirroring is impossible; documented above so a later plan can backfill if/when the sibling becomes reachable.
